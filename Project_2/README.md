Разработка документации для доработки системы управления отношениями с гостями отеля
===

*Вертикаль*: HoReCa

*Задача*: Подготовить и согласовать необходимую документацию для доработки PMS-системы отеля и ее интеграции с CRM

*Результат*: Вся необходимая для доработки документация подготовлена и согласована со стейкхолдерами

*Стек*: BPMN, UML, draw.io, Miro, Google Docs, SIPOC Map

*Артефакты*: AS IS модели (BPMN) создания бронирования и обработки претензии, ООО, ТЗ, ФТ, НФТ, Use Cases, SIPOC Proccess Map, ПМИ, UML: use case (диаграмма прецедентов), диаграммы классов и объектов
> *Все подготовленные артефакты лежат в папке [Project_2](https://github.com/ekaterinaserovaBA/PortfolioProjects/tree/main/Project_2)*

Цель и задачи проекта
--
Оценить возможность увеличить лояльность и удовлетворенность клиентов благодаря ведению базы клиентов и внедрению индивидуальных предложений.

**Задачи обследования:**
- выявить проблемы и ограничения текущих процессов в рамках системы, реализующей управление историей бронирования клиентов и маркетинговые коммуникации с клиентами,
- определить требования Заказчика к решению, выявить основные функциональные возможности, которые способны оптимизировать процессы в каждом из отделов Заказчика, участвующем в управлении историей бронирований и маркетинговых коммуникаций с клиентами,
- определить функциональность, которая должна быть реализована в рамках внедряемого решения.
