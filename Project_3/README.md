Коммерческий проект: Разработка сайта для медийного портала о женщинах в науке
===

*Задача*: Проработать цели, видение и требования к медийной платформе, направленной на популяризацию женщин-учёных, поддержку их профессионального развития и объединение сообщества.

*Результат*: Описаны требования (БТ, ФТ, НФТ) и пользовательские сценарии. Подготовлено ТЗ для реализации решения для команды разработки и lo-fi прототипы интерфейса.

*Стек*: User Story Map, draw.io, Miro

*Артефакты*: User Story Map с выделенным MVP, lo-fi прототип в Figma, ТЗ
> *Все подготовленные артефакты лежат в папке [Project_3](https://github.com/ekaterinaserovaBA/PortfolioProjects/tree/main/Project_3)*

Проблематика
=====
**Основная проблема**: у фонда есть сайт, на котором разамещена информация о проводимой им премии "Колба", а также дублирующие друг друга страница ВКонтакте и telegram-канал. При этом информация и активы у фонда есть, но они разрознены и не образуют единую экосистему: участницы не могут найти возможности, ресурсы для развития и других женщин-коллег для нетворкинга и участия в проектах.

Проработка персон и создание user stories
---
Совместно с продакт-менеджерами мы провели исследование целевой аудитории фонда и запустили опрос среди участниц, который помог сформировать 3 основные персоны и выделить целевое ядро (ту часть аудитории, которая играет центральную роль при использовании доступных ресурсов фонда).
Для этих персон я описала user stories и подготовила User Story Map, ориентируясь на MVP.

Проектирование ролевой модели для платформы
---
Поскольку перед командой стояла необходимость в проработке требований для новой платформы, а не обновления старого сайта, я выделила несколько ролей, необходимых для комфортной работы. Для каждой из ролей также были описаны user stories, которые впоследствии помогли выделить функциональные требования для дальнейшей реализации решения.
